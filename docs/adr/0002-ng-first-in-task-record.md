# TaskRecord 激活即写入 NG，全部 OK 后更新为 OK

TaskRecord 创建时 `task_result` 立即写入 `NG(0)`。所有螺栓 TighteningData 完成后，若全部 OK，则更新为 `OK(1)`。

**理由**: 防止进程崩溃或异常中断导致任务记录"消失"。工业拧紧场景中，记录丢失比时间精度偏差更严重。即使 `modify_time` 不是精确的故障时刻，库中至少有一条 NG 记录可供追溯。

**权衡**: 牺牲了 `modify_time` 对故障时刻的精确反映，换取了极端故障场景下的数据安全性和可追溯性。

**备选方案**: 任务激活时不设值，NG 时才写入。优点是时间更精确，但进程崩溃时可能丢失记录。
